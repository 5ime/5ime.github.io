记一次 Valine 被恶意刷评论

2022年03月29日 文章 约 650 字 大概 2 分钟

从昨天晚上开始到今天早上就一直被刷评论，由于博客评论配置了邮件通知所以从攻击开始我就得到了通知。邮箱疯狂推送了接近 600 条邮件

恶意评论数据

使用img标签插入了一段jsfuck编码后的代码，暂不了解意义(无限重置?)
由于评论管理页面会解析html代码，所以博主直接无法进入评论管理页面。

不过区区600条，直接前往 LeanCloud 批量删除即可，如果被刷的量非常大建议导出备份后在本地选中删除后再导入(因为 LeanCloud 一次只能删除100条。

防御方法

一定要配置评论邮件通知功能！

察觉被攻击及时前往LeanCloud关闭 Comment 的 写入权限，以免数据越来越多。

结构化数据 -> Comment -> 权限 -> 默认 ACL 权限 -> write -> 数据创建者（Owner）

其次就是先关闭博客评论，原本感觉攻击结束了，昨天晚上我又打开评论功能了。没想到趁着我睡觉又开始刷了…

写在后面

博主琢磨了半天，也没琢磨出有什么更好的防御方法，前端的一切验证都只能防君子,所以博主把博客评论系统从 Valine 替换为了 Waline。

Waline 与 Valine 最大的不同就是增加了服务端中间层，解决 Valine 暴露出来的安全问题。并且 Waline 和 Valine 的 LeanCloud 配置是可以共用的，不需要进行数据转换
